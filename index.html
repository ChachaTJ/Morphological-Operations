01 — Motivation

Why do we need morphology?

Real-world images are messy. Binary masks from thresholding have holes, noise speckles, and broken edges. Morphological operations fix this using nothing more than a geometric probe.

science

Noise Removal

Salt-and-pepper noise (isolated white pixels) is erased by erosion or opening — without a convolution kernel.

location_city

Shape Analysis

Extract boundaries, fill holes, count connected regions, and separate touching objects — all fundamental to segmentation.

grid_view

Pre/Post-Processing

Morphology cleans up thresholded images before OCR, edge detection, and object recognition — making downstream steps robust.

text_fields Text Manipulation

Dilation thickens text, Erosion thins it, Gradient extracts the outline.

Hexagons, digits, circles — each structure tests a different morphological behaviour.

02 — Method Definitions

Key terminology

Every morphological operation is built from two concepts: a Structuring Element (SE) and a comparison rule — FIT, HIT, or MISS.

grid_on Structuring Element (SE / Kernel)

A small binary matrix centered on each image pixel. Its shape and size define the neighbourhood probe. The center cell (⊕) is the anchor. Common shapes below (radius = 2):

FIT

check_circle Full Overlap

All foreground SE cells overlap with foreground image pixels.
→ Erosion outputs 1 here.

HIT

bolt Partial Overlap

At least one SE foreground cell overlaps a foreground pixel.
→ Dilation outputs 1 here.

MISS

cancel No Overlap

Zero SE foreground cells overlap any foreground pixel.
→ Both Erosion and Dilation output 0.

check_circle Where it works well

  • Binary images with clear foreground/background separation
  • Removing small noise blobs (opening)
  • Filling small interior holes (closing)
  • Extracting object boundaries (gradient = dilate − erode)
  • Separating closely touching connected objects
  • Illumination correction (Top Hat on uneven background)

warning Where it struggles

  • Grayscale images — results are less intuitive
  • Large kernel on thin features destroys structure
  • Doesn't distinguish noise from real thin structures
  • Sensitive to threshold choice before binarization
  • Repeated iterations on complex shapes can merge objects
06 — Observations & Parameters

How parameters affect results

Each knob changes the output in a predictable, understandable way. Understanding these is key to applying morphology correctly in practice.

Parameter Range Effect on Output Impact
Operation Type Erosion / Dilation / Opening / Closing / … Fundamental: shrinks vs grows vs compound effects. Choice completely changes output semantics. Critical
Kernel Shape Square, Cross, Disk, Diamond Controls directionality — square affects diagonals, cross only cardinal directions, disk is isotropic (uniform in all directions). High
Kernel Radius 1 – ∞ (typically 1–5) Larger radius = stronger, more aggressive transformation. Small features may be destroyed. r=1 is usually safe for noise removal. High
Iterations 1 – 10+ Applying N times ≈ applying with radius × N. Can fully erode thin structures in 2–3 iterations even with r=1. Amplifies
Polarity White=FG / Black=FG Swaps which pixels are "object." Erosion on dark objects on bright bg = invert polarity. Useful for inverted images. Context
Threshold (pre-step) 0 – 255 Determines what counts as foreground before binarization. Lower threshold = more white pixels → dilation appears stronger. High

lightbulb What surprised me

  • Opening ≠ Erosion: Opening selectively removes features smaller than the SE while approximately restoring the shape of larger objects — erosion just shrinks everything.
  • Top Hat illumination trick: Top Hat (original − opened) highlights bright features on a non-uniform background — solving uneven lighting without any learned model.
  • Shape matters more than size: A cross kernel radius-3 and square kernel radius-1 can produce very different edge effects even at similar computational cost.
  • Binary vs grayscale differing intuitions: On grayscale, dilation picks local maximum and erosion picks local minimum — same SE, completely different interpretation.
07 — Takeaways

Key conclusions

Morphological operations: surprisingly powerful for how simple they are.

1

Morphology = shape-aware filtering

Unlike convolution blur which averages, morphological ops answer: "does this shape FIT or HIT here?" — making them perfectly suited for binary masks and geometric analysis.

2

Two primitives, infinite combinations

Everything — Opening, Closing, Gradient, Top Hat, Skeletonize, Geodesic — is built from just Erosion and Dilation. Mastering these two gives you the whole family for free.

3

Parameter choice is domain knowledge

There is no universal "best" kernel size. The right choice depends on the scale of features to preserve vs remove — it requires application insight, not just trial-and-error.

4

Responds to: edges, blobs, thin lines

Gradient captures boundaries; Top Hat captures small bright blobs; Black Hat captures small dark blobs. It does NOT respond to gradients or textures directly — purely structural.

5

Still relevant in the deep learning era

Morphological post-processing is standard in medical imaging, document layout analysis, and satellite imagery — zero learned parameters, fully interpretable, deterministic.

import cv2, numpy as np # Structuring elements kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (5, 5)) # square kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5)) # disk kernel = cv2.getStructuringElement(cv2.MORPH_CROSS, (5, 5)) # cross # Primitives eroded = cv2.erode(img, kernel, iterations=1) dilated = cv2.dilate(img, kernel, iterations=1) # Compound operations opened = cv2.morphologyEx(img, cv2.MORPH_OPEN, kernel) closed = cv2.morphologyEx(img, cv2.MORPH_CLOSE, kernel) gradient = cv2.morphologyEx(img, cv2.MORPH_GRADIENT, kernel) tophat = cv2.morphologyEx(img, cv2.MORPH_TOPHAT, kernel) blackhat = cv2.morphologyEx(img, cv2.MORPH_BLACKHAT, kernel)
